<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner to repository main branch | Contents-write workflow commits and pushes generated static data back to the repository. |
| External scraper output to CI filesystem | Public external data and scraper code produce entity IDs, entity types, records, and timestamps written under public/data. |
| Pipeline metadata to freshness claims | `_pipeline-meta.json.lastRun` tells users and deploy automation whether a run is fresh. |
| Snapshot retention CLI to rollback artifacts | CLI args control deletion of old snapshot directories. |
| Committed public/data to browser | Static JSON becomes user-visible frontend data after deploy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Repudiation | .github/workflows/data-pipeline.yml | mitigate | Add tested `git push` after successful generated-data commit so Actions refreshes leave durable commit/snapshot history on main. |
| T-04-04-02 | Tampering | scripts/run-pipeline.ts writeEntityIfChanged | mitigate | Validate entity IDs and entity types as safe path segments before path construction; also enforce resolved path remains under `public/data/entities`. |
| T-04-04-03 | Tampering | scripts/run-pipeline.ts grouping | mitigate | Reject conflicting entity types for the same entity ID before writing, compiling, or updating metadata. |
| T-04-04-04 | Repudiation | scripts/run-pipeline.ts metadata update | mitigate | Reject all-failed, zero-record, or zero-written runs before `compile(dataDir)` and before updating `_pipeline-meta.json.lastRun`, preventing stale data from being published as fresh. |
| T-04-04-05 | Denial of Service | scripts/snapshot-data.ts retention | mitigate | Reject non-positive/non-integer/NaN retention before creating or pruning snapshots so invalid CLI input cannot delete all retained rollback snapshots. |
| T-04-04-06 | Tampering | scripts/validate-data.ts aggregate validation | mitigate | Validate ISO timestamps, history series arrays, numeric scores, confidence range, and factors objects before generated data can be committed or deployed. |
| T-04-04-07 | Information Disclosure | workflows/scripts | accept | No secrets, backend services, paid APIs, or new data exposure paths are introduced; generated data remains intended public static JSON. |
</threat_model>
