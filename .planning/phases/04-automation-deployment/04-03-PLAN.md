<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub push event to Pages workflow | Repository pushes, including generated public/data commits, trigger static site build and deployment. |
| Committed public/data to browser | Data files are bundled/served as static assets and consumed by frontend fetchers. |
| Workflow OIDC token to GitHub Pages | Deployment uses id-token and Pages permissions to publish dist. |
| Privileged Pages/OIDC workflow to third-party actions | A Pages/OIDC workflow executes downloaded GitHub Actions code before publishing the site. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Elevation of Privilege | .github/workflows/deploy.yml | mitigate | Use Pages least-privilege permissions: `contents: read`, `pages: write`, `id-token: write`; explicitly avoid `contents: write`. |
| T-04-03-02 | Tampering | deployment build | mitigate | Run `npm run data:validate` before build so malformed committed data blocks new deployment, leaving prior Pages deployment serving last valid data. |
| T-04-03-03 | Information Disclosure | deploy workflow | mitigate | No `secrets.` references and no paid API keys are needed for static Vite build. |
| T-04-03-04 | Denial of Service | Pages deploy | mitigate | Build must pass `npm test`, `npm run typecheck`, `npm run build`, and `npm run bundle:check`; failed builds do not deploy broken dist. |
| T-04-03-05 | Spoofing | GitHub Pages environment | transfer | GitHub controls Pages deployment identity through the Pages environment and OIDC; workflow uses least-privilege `id-token: write` only for Pages deployment. |
| T-04-03-06 | Elevation of Privilege | mutable action dependencies in .github/workflows/deploy.yml | mitigate | High-severity supply-chain risk: mutable tags like `actions/upload-pages-artifact@v3`, `actions/deploy-pages@v4`, `@main`, or `@master` can change under a Pages/OIDC workflow. Pin every `uses:` reference to an immutable 40-character commit SHA and enforce with contract tests rejecting major-only action tags. |
| T-04-03-07 | Denial of Service | deploy trigger filters | mitigate | Do not add deploy `paths` or `paths-ignore`; generated `public/data/**` pushes from the data pipeline must trigger deploy so users receive refreshed data after successful pipeline commits. |
</threat_model>
