<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External data sources to pipeline | Scraper responses from public APIs cross into generated JSON committed to the static site. |
| Generated JSON to browser | Static public/data files are rendered by React as data and must not become executable content. |
| CI filesystem to git repository | Automation writes generated files that may be committed by GITHUB_TOKEN. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | scripts/run-pipeline.ts | mitigate | Generate into a configurable staging directory via COMPUTEATLAS_DATA_DIR so failed runs cannot mutate served root data before validation. |
| T-04-01-02 | Tampering | scripts/validate-data.ts | mitigate | Validate required aggregate JSON shapes, finite numeric scores, confidence range, required mappings, and factor weight sum before promotion. |
| T-04-01-03 | Denial of Service | scripts/snapshot-data.ts | mitigate | Keep only 30 snapshots and delete only old snapshot directories listed in manifest; never delete root latest/rankings/history/meta/entity files. |
| T-04-01-04 | Information Disclosure | package/scripts | accept | No secrets or external credentials are introduced; all generated files are public static data by project design. |
| T-04-01-05 | Repudiation | scripts/snapshot-data.ts | mitigate | Manifest records snapshotId, createdAt, sourceDataDir, and copied file list for rollback traceability. |
</threat_model>
