<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workflow_dispatch input -> deploy shell | `inputs.source_sha` is externally supplied workflow input that controls which source tree is checked out for production Pages deployment. |
| Data Pipeline handoff -> Deploy Pages | `data-pipeline.yml` can pass a generated commit SHA to `deploy.yml`; deploy must prove the SHA is main-approved before publication. |
| Git checkout state -> Pages artifact | The checked-out source tree becomes the Vite build and Pages artifact if not rejected before build/upload/deploy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08-01 | Elevation of Privilege | `.github/workflows/deploy.yml` workflow_dispatch `source_sha` | mitigate | Add `Verify requested source SHA is on main` before build/publish; reject any `source_sha` for which `git merge-base --is-ancestor "$REQUESTED_SHA" origin/main` fails. |
| T-04-08-02 | Tampering | GitHub Pages production artifact | mitigate | Ensure non-main source cannot become `dist` by running the ancestry guard before `npm ci`, validation, build, upload, and deploy steps; contract test enforces ordering. |
| T-04-08-03 | Spoofing | Data Pipeline handoff metadata | mitigate | Treat handoff `source_sha` as untrusted until verified against `origin/main`; use env indirection rather than direct workflow-input interpolation in shell. |
| T-04-08-04 | Repudiation | Deployment auditability | mitigate | Preserve explicit failure message `Requested source SHA $REQUESTED_SHA is not reachable from origin/main` in workflow logs and keep existing source_sha HEAD equality check. |
| T-04-08-05 | Information Disclosure | Workflow logs | accept | SHA values are public repository commit identifiers; no secrets or paid API keys are introduced. |
| T-04-08-06 | Denial of Service | Deploy workflow false rejection | mitigate | Use `fetch-depth: 0` plus explicit `git fetch --no-tags --prune origin main:refs/remotes/origin/main` so legitimate main-reachable SHAs have commit graph available for `merge-base`. |
</threat_model>
