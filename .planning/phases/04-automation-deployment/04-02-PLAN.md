<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub event to workflow runner | Scheduled, manual, and push events execute repository automation with GITHUB_TOKEN. |
| External APIs to generated data | Scraper outputs from public APIs enter committed public/data artifacts. |
| Workflow runner to repository | CI commits generated data back to the repository. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Elevation of Privilege | .github/workflows/data-pipeline.yml | mitigate | Use least practical permission for this job: `contents: write` only; no `actions`, `pages`, `id-token`, or broad default permissions. |
| T-04-02-02 | Information Disclosure | .github/workflows/data-pipeline.yml | mitigate | Workflow contains no `secrets.` references and uses no paid API keys; contract test asserts absence. |
| T-04-02-03 | Tampering | data promotion step | mitigate | Generate into `public/data-staging`, validate staged data, then promote to `public/data` only after validation succeeds. |
| T-04-02-04 | Tampering | git commit step | mitigate | Commit only `public/data` with `git add public/data`; contract test blocks `git add .` and `git add -A`. |
| T-04-02-05 | Denial of Service | scheduled workflow | mitigate | Set `timeout-minutes: 15` and `concurrency.cancel-in-progress: false` so runs do not terminate an in-progress data refresh mid-write. |
| T-04-02-06 | Repudiation | automated commits | mitigate | Use fixed git identity `ComputeAtlas Bot <actions@github.com>` and commit message `data: refresh automated snapshot [skip ci]`. |
</threat_model>
