<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Static JSON -> React rendering | Entity names, risk tiers, and source-derived labels enter UI from public JSON files. |
| User keyboard/mouse -> UI state | Ranking scope and selected rows change browser-only state and must not trigger unsafe navigation. |
| App shell -> public users | Visual labels could overstate confidence/risk unless constrained by copy contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering/XSS | `RankingRail`, badges, loading states | mitigate | Render JSON strings as React text only and do not use React raw HTML injection props. |
| T-03-07 | Denial of Service | `RankingRail` | mitigate | Render loading, error, and empty states when React Query data is absent or failed. |
| T-03-08 | Spoofing/Integrity | `RiskBadge`, copy labels | mitigate | Risk badges must include `Modeling assumption` and must not contain `legal risk`, `compliance verdict`, or `sanctioned`. |
| T-03-09 | Information Disclosure | App shell | accept | All displayed data is public static index data; no secrets or user data exist in this phase. |
</threat_model>
