## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner gaps, dense table cell affordance spacing |
| sm | 8px | Compact element spacing, table row vertical padding, toolbar item gaps |
| md | 16px | Default panel padding, rail item spacing, form/filter control spacing |
| lg | 24px | Section padding, map overlay offsets, detail rail internal groups |
| xl | 32px | Layout gaps between primary map, rankings rail, and detail rail |
| 2xl | 48px | Major section breaks inside detail pages and chart stacks |
| 3xl | 64px | Page-level spacing for non-map routed detail screens |

Exceptions: minimum pointer target is 44px by 44px for icon-only controls; dense ranking rows may use 36px height on desktop only; map markers may use 12px, 16px, or 24px circles based on entity level and score bucket; 1px borders are allowed for HUD panel dividers and map outlines.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 32px | 600 | 1.1 |

Typography rules:
- Use exactly two font weights in UI implementation: 400 regular and 600 semibold.
- Use tabular numbers for scores, ranks, timestamps, confidence, factor values, and chart axis labels.
- Labels are uppercase with 0.08em letter spacing only for HUD section labels, badges, map legends, and table headers.
- Body copy remains sentence case and must not use all-caps for explanatory paragraphs.
- Display text is reserved for the selected entity score, page title metric, or one primary dashboard KPI only.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #060A12 | App background, map underlay, page shell, modal scrim |
| Secondary (30%) | #0B1120 | Glass panels, ranking rail, detail rail, navigation, chart containers |
| Accent (10%) | #22D3EE | Selected entity outlines, active filter/nav states, map hover/click target, focused controls, primary CTA, links, loading sweep |
| Destructive | #EF4444 | Destructive actions only; Phase 03 has no destructive user actions |

Accent reserved for: selected map features, active breadcrumbs, active entity type filters, focused keyboard state, primary CTA, links, loading/progress indicators, and the currently selected ranking row. Do not use cyan for passive decoration or every border.

Additional semantic data colors within the 10% non-neutral budget:
- Positive/growth/capacity signal: #22C55E, reserved for positive score change, high-growth trend segments, and high-score map ramp stops.
- Warning/sparse/stale/risk assumption: #F59E0B, reserved for low confidence, partial completeness, stale freshness, and risk-modeling assumption badges.
- Muted/inactive data: #64748B, reserved for de-emphasized low-confidence rows, unavailable factors, and disabled filters.

Color application rules:
- Use translucent glass panels as `rgba(11, 17, 32, 0.78)` with blur only when text contrast remains readable.
- Panel borders use `rgba(34, 211, 238, 0.18)` by default and `rgba(34, 211, 238, 0.42)` only for selected/focused panels.
- Map choropleth ramp uses dark navy for no/zero score, cyan for mid-score, green for high score, and amber overlay indicators for sparse confidence; never use red for export-control risk unless the UI labels it as a modeling assumption.
- Text contrast target: primary text #E5F0FF on dominant/secondary surfaces; secondary text #94A3B8; disabled text #475569.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Explore Index |
| Empty state heading | No compute signals found |
| Empty state body | Try a broader entity level, clear filters, or select another region on the map. Sparse entities remain visible when a usable trend signal exists. |
| Error state | Compute data could not load. Check the static data files, then retry or continue with the last committed dataset. |
| Destructive confirmation | None — Phase 03 exposes no destructive user actions. |

Copywriting rules:
- Describe the index as a predictive trend signal, not an exact capacity census.
- Confidence copy must be contextual and compact: `Confidence 4/5`, `Partial factors`, `Updated 3d ago`.
- Risk copy must say `Modeling assumption` or `Risk adjustment`, never `legal risk`, `compliance verdict`, or `sanctioned`.
- Company overlay copy must say `corporate compute influence` or `AI CapEx signal`; do not imply exact facility locations.
- Missing data copy must not apologize repeatedly. Prefer direct state labels: `Factor unavailable`, `No recent history`, `Partial source coverage`.

## Interaction Contract

### Map-First Shell

- The initial route opens to a full-viewport 2D MapLibre map with country choropleth as the visual anchor.
- The map must render before ECharts trend panels. Trend and detail chart code is lazy-loaded after entity selection or routed detail navigation.
- Keep one compact ranking/detail rail visible on desktop so comparison is possible without leaving the map.
- The map shell layout is desktop-first: top status/header strip, primary map canvas, left or right ranking rail, and collapsible detail rail.
- On narrow screens, stack rails below the map and preserve selection, breadcrumbs, and shareable hash route state.

### Drill-Down Behavior

- Country click is zoom-first: click a country, fly/zoom the map to the country/region, set it as selected, then reveal city/cloud-region next steps in the detail rail.
- Spatial hierarchy is country fill → city markers → cloud-region markers. Company overlay is separate and symbolic.
- Breadcrumb order is `Global / Country / City / Cloud Region / Company` when applicable; company may appear from overlay without implying a physical child of a cloud region.
- Hash routes must remain shareable and GitHub Pages compatible. Route state should reflect selected entity type and entity id where practical.
- Selecting a ranking row must pan/zoom/highlight the corresponding map feature and update the detail rail.

### Rankings and Tables

- Rankings must exist for countries, cities, and cloud regions, sorted by index score descending by default.
- Ranking rows show rank, entity name, index score, score change when available, confidence badge, completeness badge, and freshness age.
- Low-confidence rows remain visible but de-emphasized with muted text and a warning badge; do not hide them by default unless score is zero or unusable.
- Tables use dense desktop rows, sticky headers, tabular numeric columns, and keyboard focus states.

### Entity Detail Pages and Rails

- Details show selected entity score, trend summary, factor breakdown, source coverage, confidence, completeness, freshness, and risk adjustment.
- Factor breakdown must separate raw factor value, normalized factor score, and factor weight.
- Risk tier and multiplier are labeled as modeling assumptions and must include non-alarmist explanatory text.
- Detail pages are routeable through HashRouter but should not replace the map-first exploration flow; routed details include a return-to-map affordance.

### Charts

- ECharts trend charts are used for selected entity detail pages and drill-down panels only, not as a blocking initial-shell dependency.
- Trend charts use dark theme styling, cyan primary line, green positive change segments or annotations, amber confidence/staleness markers, and muted grid lines.
- Empty chart state uses the copywriting contract and leaves the rest of the entity detail usable.

### Loading, Empty, and Error States

- Initial map loading: show dark shell immediately with cyan loading sweep and status label `Loading compute index`.
- Data loading within panels: use skeleton rows/cards that preserve layout dimensions.
- Empty states must identify the current filter/entity scope and provide one next action.
- Errors must distinguish static data load failure from empty data and provide a retry or fallback path.

### Accessibility and Focus

- Every icon-only control must have an accessible label and 44px minimum pointer target.
- Keyboard focus is cyan outline with at least 2px visible stroke and no layout shift.
- Map interactions must have non-map equivalents through rankings, breadcrumbs, and detail rail controls.
- Do not rely on color alone for confidence, freshness, growth, or risk; include text badges or labels.
