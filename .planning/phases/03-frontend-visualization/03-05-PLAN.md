<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Static GeoJSON/TopoJSON + index JSON -> MapLibre | Public static data controls map feature properties and marker labels. |
| Map click events -> Zustand/hash route | Untrusted browser events update UI state and shareable hash URLs. |
| Derived cluster proxy labels -> user interpretation | Modeled cluster proxy data can be mistaken for verified facility data if copy is unclear. |
| Company data -> geographic overlay | Company data can be misrepresented as exact facility location if labels are wrong. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | `country-join.ts` | mitigate | Join via explicit ISO/name metadata and default unknown features to `hasComputeSignal: false` and `score: 0`. |
| T-03-15 | Denial of Service | `ComputeMap.tsx` | mitigate | Render loading/error states when geometry, latest index, or crossref queries are unavailable. |
| T-03-16 | Spoofing/Integrity | `drilldown.ts`, `ComputeMap.tsx` | mitigate | Company overlay strings must contain `corporate compute influence` or `AI CapEx signal` and `not an exact facility location`. |
| T-03-17 | Spoofing/Integrity | cluster proxy UI | mitigate | Cluster strings must contain `modeled cluster proxy` and `not a verified facility`; route/detail support must not require real cluster entity files. |
| T-03-18 | Tampering | Hash route updates | mitigate | Only write hash routes from known entity levels and selected IDs from static data/crossref/derived cluster ids. |
</threat_model>
