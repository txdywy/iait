<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hash route params -> detail lookup | URL hash contains entity type/id selected by users or shared links. |
| Static index/history/config JSON -> detail/chart rendering | Public JSON drives detail text, factor rows, and chart series. |
| Dynamic chart chunk -> browser runtime | Lazy-loaded chart bundle can fail after deployment or cache mismatch. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-15 | Tampering | `EntityDetailRoute.tsx` route params | mitigate | Validate `type` against known entity types and show `No compute signals found` when `id` is absent from `latest.entities`. |
| T-03-16 | Tampering/XSS | `FactorBreakdown`, `TrendChart` labels | mitigate | Render factor/config/history strings as normal JSX text and never as raw HTML. |
| T-03-17 | Spoofing/Integrity | `RiskBadge`, detail risk copy | mitigate | Include `Modeling assumption` and forbid `legal risk`, `compliance verdict`, and `sanctioned` in detail copy. |
| T-03-18 | Denial of Service | Lazy ECharts chunk | mitigate | Wrap chart import in `Suspense` fallback `Loading trend chart...`; app shell remains usable if chart data is empty. |
| T-03-19 | Denial of Service | Initial bundle | mitigate | Run `npm run bundle:check` after build; ECharts imports must not appear in app/router/main shell files. |
</threat_model>
