## Files Created/Modified

- `scripts/prepare-geo.ts` - Copies the world-atlas country TopoJSON asset and entity cross-reference mapping into public static data paths.
- `public/data/geo/countries-110m.json` - Static country boundary TopoJSON for downstream MapLibre country rendering.
- `public/data/entity-crossref.json` - Browser-fetchable copy of the entity hierarchy and provider/country/city mapping.
- `scripts/check-initial-bundle.mjs` - Gzips Vite entry chunks and fails if any initial JavaScript entry exceeds 300KB gzip.
- `public/data/latest.json` - Aggregate latest entity index snapshot generated by the pipeline/compiler.
- `public/data/rankings.json` - Aggregate ranking snapshot generated by the pipeline/compiler.
- `public/data/history.json` - Aggregate score history snapshot generated by the pipeline/compiler.

## Decisions Made

- Used public static asset copying rather than importing `world-atlas` JSON from application code, preserving the Phase 3 bundle-size goal and allowing browser consumers to fetch geography by URL.
- Committed only the plan-scoped aggregate outputs (`latest.json`, `rankings.json`, `history.json`) even though `npm run pipeline` also generated per-entity files and pipeline metadata; those additional generated files were outside the strict allowed implementation file list.

None - plan executed exactly as written.

## Issues Encountered

- `npm run pipeline` generated additional per-entity JSON files and `_pipeline-meta.json`. These were not committed because the user-provided strict scope allowed only the aggregate JSON outputs for this plan.
- The committed aggregate JSON snapshots contain generation timestamps. A full verification pipeline run refreshed them during testing, so they were restored to the committed Task 2 snapshot afterward to keep the worktree clean without adding an extra task commit.
