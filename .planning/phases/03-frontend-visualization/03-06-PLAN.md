<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hash route params -> detail lookup | URL hash contains entity type/id selected by users or shared links. |
| Static index/history/config JSON -> detail/chart rendering | Public JSON drives detail text, factor rows, and chart series. |
| Derived cluster id -> modeled detail panel | Cluster ids are derived from cloud-region crossref, not real aggregate entities. |
| Dynamic chart chunk -> browser runtime | Lazy-loaded chart bundle can fail after deployment or cache mismatch. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-19 | Tampering | `EntityDetailRoute.tsx` route params | mitigate | Validate `type` against known entity levels and resolve `data-center-cluster` through `resolveClusterDetail`; show `No compute signals found` for unknown ids. |
| T-03-20 | Tampering/XSS | `FactorBreakdown`, `TrendChart` labels | mitigate | Render factor/config/history strings as normal JSX text and never as raw HTML. |
| T-03-21 | Spoofing/Integrity | cluster detail copy | mitigate | Include `modeled cluster proxy` and `not a verified facility`; never fetch or imply verified cluster facility files. |
| T-03-22 | Spoofing/Integrity | `RiskBadge`, detail risk copy | mitigate | Include `Modeling assumption` and forbid `legal risk`, `compliance verdict`, and `sanctioned` in detail copy. |
| T-03-23 | Denial of Service | Lazy ECharts chunk | mitigate | Wrap chart import in `Suspense` fallback `Loading trend chart...`; app shell remains usable if chart data is empty. |
| T-03-24 | Denial of Service | Initial bundle | mitigate | Run `npm run bundle:check` after build; ECharts imports must not appear in app/router/main shell files. |
</threat_model>
