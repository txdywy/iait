<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> static JSON assets | Browser fetches public JSON/TopoJSON generated by pipeline or copied from dependencies; malformed files must not crash the app silently. |
| Build config -> deployed static site | Vite base path and public asset paths determine what GitHub Pages serves; wrong configuration can break all data loading. |
| Frontend bundle -> public users | All code and environment variables are visible to users; no secrets can be introduced. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | `vite.config.ts`, `src/data/static-json.ts` | mitigate | Do not read or define frontend secrets; only fetch relative `data/*.json` paths via `import.meta.env.BASE_URL`. |
| T-03-02 | Denial of Service | `src/data/static-json.ts` | mitigate | Throw descriptive errors for non-OK static fetch responses and let React Query consumers render error states in later plans. |
| T-03-03 | Tampering | `src/data/types.ts`, `src/data/queries.ts` | mitigate | Keep frontend types aligned with compiler outputs and add tests for failed loads instead of assuming static JSON is always present. |
| T-03-04 | Denial of Service | `scripts/check-initial-bundle.mjs` | mitigate | Fail `npm run bundle:check` when the initial JS entry chunk exceeds 300KB gzip. |
| T-03-05 | Information Disclosure | `package.json`, Vite env handling | accept | Phase 3 has no auth, API keys, or private data; public static JSON is intentionally served. |
</threat_model>
